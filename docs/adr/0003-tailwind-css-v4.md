# Tailwind CSS v4 in Frontend

## Context and Decision

The web application development guidelines suggest using Vanilla CSS for styling. However, the existing `App.tsx` file was already implemented using Tailwind CSS utility classes. To avoid the time-consuming and error-prone process of rewriting the entire UI in Vanilla CSS within the hackathon timeline, we decided to install and configure Tailwind CSS v4. This allows us to keep the existing component code intact, leverage the modern CSS-first configuration of Tailwind v4, and ensure high styling speed for the remaining frontend work.
